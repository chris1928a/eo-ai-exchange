What I Do Not Do

Four anti-patterns I avoid.

×
No Vector-DB over personal brain
Threshold is ~10K files. I have ~200. Anthropic, Y Combinator, Karpathy all dropped vectors for agentic search. ripgrep returns in <50ms.
×
No Drive as brain storage
API rate limits, 200 to 2000ms latency vs <50ms ripgrep, OAuth refresh issues. Drive is for source-of-truth originals, not for a brain.
×
No OneDrive plus Git
Git index corruption risk. OneDrive syncs .git/objects while Git writes. GitHub Desktop warns explicitly. Fix: moved repo to local C:\Code.
×
No mega-CLAUDE.md with everything
Garry Tan had 20.000 lines, model attention degraded. Cut to ~200 lines of pointers. My CLAUDE.md is ~50 lines. Skills load on demand.

SourcesBoris Cherny (Anthropic, "Claude Code uses agentic search, not vectors"), Karpathy (LLM Wiki Pattern, April 2026), Garry Tan (gbrain repo)

What Did Not Work

Audit is routine, not failure.

01
Drive-Hub Phase 2 half-finished. 39-day memory lag, Drive-API rate limits killed sync. Decision: kill the Drive-Hub idea, use Git-Pull from GitHub mirror as sync mechanism.
Resolved 27.04.
02
175 uncommitted files from 30 days of build-up. Hygiene debt that obscured what was active and what was archive. Cleaned up via Robocopy + Git reset.
Resolved
03
OneDrive + Git race condition. Git index corruption risk. Repo moved from OneDrive sync path to local C:\Code in 43 seconds.
Resolved
04
Memory bloat: 54 SACAA exam memos stayed in active index after passing the test. Index file consolidation reduced active memory from 124 to 60 lines.
Resolved

Garry Tan's discipline applies to my own setup too. If I have to ask my system for the same thing twice, I failed.
