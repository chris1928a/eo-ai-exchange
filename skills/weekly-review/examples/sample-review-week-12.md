## What this output reveals about the system maturity

1. **Trend detection working.** The last paragraph is the magic, "3rd week running" only fires after 4+ weeks of archived reviews to compare against.
2. **Decay flags have ages.** "8 days late", "14 days late", these come from the open_threads memory file having timestamps.
3. **Wins cite numbers.** "+18%", "12 stars in 48h", "3.2k impressions", not generic "good progress".
4. **Decisions have dates.** Every decision has a (YYYY-MM-DD) for auditability.
5. **Top 3 next week are commitments.** Each one has a verifiable outcome, not a wish.
6. **Single weekend decision is binary.** "Pick one, do not split", forces clarity.

If your week-12 review still looks like a generic gratitude list, your memory files aren't accumulating. Run `/memory-curator` and check that hats and open_threads files are growing.
