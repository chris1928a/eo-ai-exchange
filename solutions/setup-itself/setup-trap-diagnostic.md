# Setup Trap Diagnostic, the 1-Pager

**Pain cluster:** Setup itself + foundational across all clusters
**Status:** Released, Event #1, 2026-05-11
**Time to use:** 5 minutes

> Print this. Pin it. Re-run it every 90 days. The 3 questions that distinguish a productive AI setup from a Setup Trap.

---

## The 3 questions

### Question 1
**Pflegst du dein Setup mehr, als du es nutzt?**
*(Do you spend more time maintaining your setup than using it?)*

If you spend > 20% of your AI-related time on setup, configuration, tool-evaluation, or migration, you are in the Setup Trap.

> **Test:** track 7 days. Time „producing output with AI" vs „configuring AI". If maintenance > 20%, this is the trap.

---

### Question 2
**Wenn dein Setup morgen stirbt, verlierst du Wissen oder nur Workflow?**
*(If your setup dies tomorrow, do you lose knowledge or just workflow?)*

If you lose KNOWLEDGE (decisions, context, accumulated insight, project history), your setup conflates Memory with Tools. That is the Setup Trap.

> **Test:** imagine your laptop dies and you must rebuild from scratch. What do you lose forever vs what is just inconvenient? Forever-loss = trap.

---

### Question 3
**Würde dein 80-jähriges Ich in 30 Jahren 2 Sätze über dein Setup verstehen?**
*(Would your 80-year-old self in 30 years understand 2 sentences about your setup?)*

If your setup requires explaining brand names, tool combinations, or vendor-specific workflows, it is overfit to 2026 and will be useless in 2030.

The right answer for your future self is something like „I had a memory layer of important decisions and a tool layer that helped me act on them. The tools changed, the memory persisted." If you cannot say something this simple, it is the trap.

---

## How to escape the Setup Trap

If you answered YES to ANY of the 3 questions, do these 3 things in order:

1. **Stop adding tools.** No new evaluations, no new subscriptions, no new tabs. 30 days minimum.
2. **Build CLAUDE.md.** Spend 30 min writing your persistent context (see [30-Min aiOS Blueprint](30-min-aios-blueprint.md)).
3. **Pick ONE workflow.** Use only your existing tools. Make ONE workflow rock-solid before touching anything else (see [60-Day Founder Onboarding](../time-learning/60-day-founder-onboarding.md)).

After 30 days, re-run the 3 questions. You should answer NO to all 3.

---

## The Memory > Tools > Models hierarchy

The Setup Trap exists because most people build the wrong way around:

| Most people build | Reality 2026 |
|---|---|
| Models → Tools → Memory | Memory → Tools → Models |
| Pick the best model first | Build memory first, models swap easily |
| Pick the best tool second | Tools serve memory, not the other way |
| Pretend memory is the tool | Memory is its own layer (CLAUDE.md, skills, docs) |

If your memory layer survives a model change OR a tool change, you are out of the trap. If it does not, you are in.
